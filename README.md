## ğŸ¯ Overview

The **PDF RAG Application** allows users to bridge the gap between their private documents and LLMs. Instead of feeding massive amounts of text into a prompt, this system:
1.  **Chunks** your PDF into manageable segments.
2.  **Embeds** those segments into vectors.
3.  **Indexes** them in a high-performance database.
4.  **Retrieves** only the most relevant context for the AI to answer your questions accurately.

---

## ğŸ—ï¸ Infrastructure (The Core Pillars)

### ğŸ—„ï¸ Qdrant: The Vector Brain
Unlike traditional SQL databases, **Qdrant** is optimized for the "nearest neighbor" searches required for AI.
*   **High-Dimensional Storage:** Specifically tuned to store and index embeddings generated by Gemini.
*   **Collection-Based Logic:** Every PDF is treated as a unique collection, allowing for isolated and precise document querying.
*   **Lightning-Fast Retrieval:** Uses HNSW indexing to find relevant context in milliseconds, ensuring the AI response is near-instant.

### ğŸ‹ Docker: Environment Consistency
The entire data layer is abstracted using **Docker**. 
*   **One-Command Setup:** No need to install Qdrant locally; `docker-compose` handles the engine, storage, and networking.
*   **Persistence:** Configured with Docker Volumes to ensure your indexed PDF data survives container restarts.
*   **Scalability:** Ready to be deployed into production environments (Kubernetes/Cloud) with zero configuration changes.
